# Vocotype Web UI 设计文档

## 概述

为 Vocotype 语音转写工具构建基于 NiceGUI 的 Web UI，替代现有的 CLI 菜单交互，面向产品/Demo 展示场景，要求专业美观。

Web UI 作为本地控制面板运行在 localhost，音频采集仍走后端（sounddevice / UDP），不走浏览器 WebRTC。现有 `app/` 核心模块保持不变，Web UI 层调用它们。

## 技术栈

- **NiceGUI** — Python Web 框架，底层 FastAPI + Vue/Quasar
- 纯 Python 编写前后端，无需 JS/TS
- 利用 NiceGUI 内置 WebSocket 通道做实时推送（转写结果、状态更新）

## 文件结构

```
app/
  web/
    __init__.py          # NiceGUI app 入口，侧边栏布局框架
    pages/
      transcribe.py      # 语音转写页面
      esp32.py           # ESP32 无线麦克风页面
      speakers.py        # 声纹管理页面
      settings.py        # 设置页面
    components/
      sidebar.py         # 侧边栏导航组件
      transcript.py      # 转写结果显示组件（文档流式）
      audio_controls.py  # 录音控制组件（按钮 + VAD 指示）
    state.py             # 全局状态管理（录音状态、转写队列、连接状态）
```

## 启动方式

- `python main.py --web` — 启动 Web UI（NiceGUI 服务器）
- `python main.py` — 保留原有 CLI 菜单（向后兼容）

## 整体布局

侧边栏导航，左侧固定菜单 + 右侧主内容区：

- 侧边栏（约 200px 宽）：
  - 顶部：品牌 Logo + 名称 "Vocotype"
  - 导航项：语音转写、ESP32 麦克风、声纹管理
  - 底部：设置入口
- 主内容区：根据当前导航项渲染对应页面

## 页面设计

### 1. 语音转写页（主页，默认页）

**音频源选择**（顶部）：
- chip/pill 按钮组：电脑麦克风 / ESP32 UDP / 其他设备
- 选中项高亮，点击切换
- 选择"其他设备"时弹出设备列表供选择

**录音控制**（音频源下方）：
- 按钮模式：大圆形录音按钮，未录音时蓝色，录音中红色 + 脉冲动画
- VAD 自动模式：显示实时音频电平条，检测到语音时高亮，无需手动操作
- 模式切换在设置页完成，转写页显示当前模式的对应控件

**转写结果区**（主体，文档流式）：
- 每条转写结果一行，包含：
  - 说话人彩色标签（如启用声纹识别）
  - 时间戳
  - 置信度徽标（小圆角标签）
  - 转写文本
- 新结果从底部追加，自动滚动到最新
- 可手动上翻查看历史，上翻时暂停自动滚动

**底部状态栏**：
- 录音时长
- 转录统计：已完成 / 已提交 / 队列剩余
- 当前模式标识（按钮/VAD、声纹模式）

### 2. ESP32 无线麦克风页

**连接配置**：
- ESP32 IP 地址输入框（默认 192.168.4.1）
- 端口输入框（默认 6000）
- 连接 / 断开按钮

**状态面板**：
- 连接状态指示（已连接 / 未连接 / 连接中）
- 实时音频电平显示

**录音保存**：
- 开关：是否保存录音到文件
- 保存路径显示

### 3. 声纹管理页

**已注册说话人列表**：
- 每行：头像色块 + 名称 + 注册时间 + 样本数
- 操作按钮：删除

**新建注册**：
- 输入说话人名称
- 设置采集样本数（默认 5，最少 3）
- 开始采集按钮 → 进入实时注册流程

**实时注册流程**：
- 进度条显示采集进度（已采集 x/n 样本）
- 每采集一个样本有视觉反馈（进度条前进 + 提示）
- 完成后自动返回列表，显示成功提示

### 4. 设置页

**录音模式**：按钮模式 / VAD 自动模式 切换

**声纹识别**：
- 模式选择：关闭 / 识别 / 过滤 / 实时注册
- 过滤模式：白名单说话人多选

**VAD 参数**：
- 语音检测灵敏度（阈值滑块）

**输出配置**：
- 追加换行开关
- 去重开关

## 视觉风格

**配色**：
- 背景：深蓝黑（#16161e 主区域，#1a1a2e 侧边栏/卡片）
- 强调色：蓝紫色 #646cff
- 录音状态：红色 #ef4444
- 成功/在线：绿色 #4ade80
- 说话人色板：#4ade80, #60a5fa, #f97316, #a78bfa, #fb7185, #fbbf24...

**字体**：系统无衬线体，中文优先 Microsoft YaHei UI

**深色主题**：全局深色，不提供浅色模式切换（Demo 场景深色更有技术感）

## 交互细节

- **录音按钮**：点击切换录音状态，录音中脉冲动画
- **VAD 模式**：音频电平实时可视化，语音段高亮
- **转写结果**：新结果滑入动画，自动滚动，手动上翻时暂停自动滚动
- **声纹注册**：进度条 + 每样本完成视觉反馈
- **页面切换**：侧边栏当前项高亮

## 状态同步

- 后端通过 NiceGUI 的 `ui.timer` 或事件推送实时更新 UI
- 录音状态、转写结果、连接状态都实时反映到界面
- 页面刷新后恢复当前状态（如正在录音，刷新后仍显示录音中）
- 全局状态通过 `app/web/state.py` 集中管理

## 与现有代码的关系

- `app/web/` 是纯新增目录，不修改现有模块
- Web UI 通过导入 `app/` 下的现有类来使用核心功能：
  - `TranscriptionWorker` / `VadTranscriptionWorker` — 转写
  - `AudioCapture` / `UDPAudioSource` — 音频采集
  - `SpeakerProcessor` / `SpeakerDB` — 声纹识别
  - `load_config` — 配置加载
- `main.py` 新增 `--web` 参数分支，调用 `app/web` 启动 NiceGUI 服务器
- 原有 CLI 流程完全保留不变

## 不在范围内

- 浏览器端音频采集（WebRTC）— 仅用后端 sounddevice/UDP
- 浅色主题
- 用户认证/多用户
- 移动端适配
