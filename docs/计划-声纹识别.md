# 声纹识别集成 — 实施计划 + 知识点

## 目标
在现有 VAD 语音转录系统中集成 CAM++ 声纹识别，支持说话人识别（标注）和说话人过滤（只转录白名单）两种模式。

## 基线
- 基线 commit: `1a28bf4` (docs: add speaker recognition design spec)
- 设计文档: `docs/设计-声纹识别.md`
- 模型: CAM++ (`iic/speech_campplus_sv_zh-cn_16k-common`), 7.2M参数, 192维embedding

## 使用方法

### 进入项目环境

```bash
cd D:\MVP\vovotyoe\vocotype
.venv\Scripts\activate
```

### 启动程序

```bash
python main.py
```

统一主菜单：
```
==================================================
  === Vocotype ===

  选择功能:
  [1] 语音转写
  [2] ESP32 无线麦克风
  [3] 声纹注册/管理

  输入 1/2/3:
```

### 功能 [1] 语音转写

选 `1` 后进入原有的转写流程：
```
选择音频源:
[1] 电脑麦克风（默认设备）
[2] ESP32 UDP
[3] 选择其他音频设备

选择模式:
[1] F2 热键模式（按 F2 开始/停止录音）
[2] VAD 自动检测模式（持续监听）

声纹识别:
[1] 关闭（不使用声纹）
[2] 识别模式（标注说话人）
[3] 过滤模式（只转录指定人）
```

快捷方式（跳过菜单直接进转写）：
```bash
python main.py --udp 192.168.4.1:6000
```

### 功能 [2] ESP32 无线麦克风

**前提**：ESP32 开发板已上电，电脑 WiFi 已连接 `ESP32-MIC`（密码 12345678）

选 `2` 后：
```
ESP32 无线麦克风模式
是否保存录音文件?
[1] 仅实时播放
[2] 播放 + 保存 recording.raw

→ 等待 ESP32 beacon → HELO 握手 → 实时播放到 VB-Cable/扬声器
→ Ctrl+C 停止
```

自动检测 VB-Cable 虚拟音频设备，路由后任何应用（微信、腾讯会议等）可将 ESP32 当作麦克风。

### 功能 [3] 声纹注册/管理

选 `3` 进入子菜单：
```
=== 声纹注册/管理 ===

[1] 注册说话人
[2] 查看已注册
[3] 删除说话人
[4] 返回主菜单
```

**注册说话人（多样本循环）**：
```
输入说话人姓名: 张三
录制音频源: [1] 电脑麦克风（默认设备）  [2] ESP32 UDP  [3] 选择其他音频设备
录制时长 (秒, 默认5):

--- 第 1 个样本 ---
Recording 5 seconds... Speak now!
已注册: 张三 (1 个样本)
录制更多样本? (y/n): y

--- 第 2 个样本 ---
Recording 5 seconds... Speak now!
已注册: 张三 (2 个样本)
录制更多样本? (y/n): y

--- 第 3 个样本 ---
...
注册完成: 张三 (共 3 个样本)
```

建议 3~5 个样本，每次说不同内容，提高识别准确率。完成后自动回到子菜单，选 `4` 返回主菜单可继续选择转写。

**独立 CLI（仍可使用）**：
```bash
python -m app.speaker_enroll enroll --name "你的名字" --source mic --duration 10
python -m app.speaker_enroll enroll --name "你的名字" --audio recording.wav
python -m app.speaker_enroll list
python -m app.speaker_enroll remove --name "你的名字"
```

### 推荐使用流程

1. `python main.py` → `3` → 注册声纹（电脑麦克风录 3 次）→ 返回主菜单
2. `python main.py` → `1` → 选音频源 → VAD 模式 → 过滤模式 → 选白名单 → 只转录自己的话

### 输出格式
- 识别模式: `[张三] 今天下午三点开会`
- 过滤模式: 正常文本（非白名单的说话人直接丢弃，不转录）
- 声纹关闭: 正常文本（和以前一样）

## 关键知识点

### 1. 模型加载方式
**不能用 `modelscope.pipelines`**，会引发 `datasets` / `addict` 等依赖版本冲突。

实际使用的方式：
```python
from modelscope.hub.snapshot_download import snapshot_download
from modelscope.models.audio.sv.DTDNN import SpeakerVerificationCAMPPlus

model_dir = snapshot_download(model_id, local_files_only=True)
# 读取 configuration.json 获取 model_config 和 pretrained_model
model = SpeakerVerificationCAMPPlus(model_dir=..., model_config=..., device="cpu", pretrained_model=...)
```

模型缓存路径: `~/.cache/modelscope/hub/models/iic/speech_campplus_sv_zh-cn_16k-common/`
权重文件: `campplus_cn_common.bin` (26.7MB)

### 2. 音频归一化（关键！）
**必须在 `extract_embedding()` 中做 peak 归一化**，否则低音量麦克风（笔记本内置麦克风 peak 仅满量程 1.5%）会导致 embedding 质量差，无法匹配。

```python
peak = np.max(np.abs(audio_f32))
if peak > 1e-6:
    audio_f32 = audio_f32 / peak * 0.95
```

### 3. 注册与使用必须同源
注册声纹的音频源必须和运行时一致：
- 用 ESP32 运行 → 用 ESP32 注册
- 用电脑麦克风运行 → 用电脑麦克风注册

不同麦克风的频响特性差异会显著降低匹配分数。

### 3.1 蓝牙/USB设备选择
选择「[3] 选择其他音频设备」会列出 Windows 已识别的所有输入设备（蓝牙麦克风、USB 麦克风、外接声卡等）。
蓝牙设备需先在 Windows 蓝牙设置中配对连接，vocotype 才能看到。
注意：注册声纹和运行转写时应选择同一个设备（参见第 3 点）。

### 4. 阈值设置
- 默认阈值: **0.45**（非论文中的 0.65）
- 实测同一人不同段音频 score ≈ 0.5~0.7
- 不同人 score ≈ -0.1~0.2
- 阈值太高会导致自己也被识别为 unknown
- 可通过 config.json 中 `speaker.threshold` 调整

### 5. 多样本注册
多次注册同一个名字，embedding 会自动累加，centroid（质心）会重新计算。建议 3~5 个样本，每次说不同内容，提高鲁棒性。

### 6. 数据流位置
声纹识别插入在 **VAD 检测语音段之后、ASR 转录之前**：
```
音频帧 → VAD 检测 → _submit_speech() → 声纹判断 → ASR 转录
                                          │
                                    过滤模式: 非白名单 → 丢弃，跳过ASR（省计算）
                                    识别模式: 附加 speaker_id → ASR → [名字] 文本
```

### 7. 安全退化
- 声纹模型加载失败 → 警告日志，自动退化为无声纹模式
- `speaker_db.json` 不存在 → 自动创建空库
- 过滤模式无已注册说话人 → 退化为识别模式
- embedding 提取失败（音频太短） → 返回 unknown，不阻塞转录

### 8. 新增依赖
- `torchaudio` (CPU版，用于 Kaldi fbank 特征提取)
- `addict`, `datasets` (modelscope 间接依赖)
